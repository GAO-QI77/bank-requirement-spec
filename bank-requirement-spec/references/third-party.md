# 第三方合作业务域扩展包（V4.0）

> 反过拟合原则：本扩展把「代收付/水费」从唯一场景降级为众多子类之一。任何具体机构、交易码、清算周期、手续费比例，原文没给一律用占位符。

## 1. 第三方合作机构类型库（第 0 轮）

第 0 轮识别合作机构属于哪一类。这影响后续监管口径、清算方式、数据敏感度。

| 机构大类 | 典型机构 | 关键特征（影响需求的维度） |
|---|---|---|
| 公共事业 | 水、电、燃气、供热 | 按欠费等额缴纳为主、批量对账、清算 T+N、有夜间停服窗口 |
| 政务机构 | 社保、医保、公积金、税务、财政 | 监管强、数据极敏感、常有专线/专用报文、对账要求严 |
| 金融机构 | 保险、基金、证券 | 涉适当性/双录/代销、资金清算复杂、监管报送多 |
| 民生服务 | 学校、医院、物业、停车、交通出行 | 小额高频、商户号管理、退费投诉多 |
| 经营类合作方 | 企业客户、平台方、商户、供应链核心企业 | 归集/代发为主、协议个性化、手续费分润复杂 |
| 其他 | 任何需查询/缴费/扣款/代发/归集/清算/对账/数据报送的外部机构 | 按实际业务套用上述维度 |

★ 第 0 轮输出"合作机构类型"时，必须引用原文证据；原文未明确机构性质的，列入待确认问题。不得默认按"水费"处理。

## 2. 第三方业务类型库（第 0 轮）

把"代缴费"扩展为完整的第三方合作业务谱系。第 0 轮需逐条勾选本需求涉及哪些，未涉及的不写。

| 业务类型 | 资金流向 | 是否涉账务 | 是否涉清算 | 高频遗漏点 |
|---|---|---|---|---|
| 代收业务 | 客户→银行→机构 | 是 | 是 | 清算周期、手续费向谁收 |
| 代付业务 | 机构→银行→客户 | 是 | 是 | 退票处理、收款账户校验 |
| 批量扣款 | 客户→机构 | 是 | 是 | 部分成功、文件级幂等、重复扣款 |
| 批量入账 | 机构→客户 | 是 | 是 | 入账失败挂账、账户状态校验 |
| 费用缴纳 | 客户→机构 | 是 | 是 | 等额/预存规则、多缴退款 |
| 退款退费 | 机构→客户 | 是 | 是 | 反向类高敏感、原路/异路、跨日 |
| 签约代扣 | 客户→机构 | 是 | 是 | 客户授权协议、解约、授权失效 |
| 账户验证 | 无 | 否 | 否 | 仅验证不动账、敏感信息、留痕 |
| 身份核验 | 无 | 否 | 否 | KYC、个人信息保护、授权 |
| 信息查询 | 无 | 否 | 否 | 数据权限、脱敏、访问留痕 |
| 数据采集 | 无 | 否 | 否 | 采集目的、最小化、保存期限 |
| 数据回盘 | 无 | 可能 | 是 | 回盘延迟/重复/格式、对账依赖 |
| 对账文件交换 | 无 | 否 | 是 | 文件格式、对账周期、不平处理 |
| 清算资金划转 | 内部户→机构 | 是 | 是 | 待清算户、清算时点、划转失败 |
| 手续费计收 | 客户/机构→银行 | 是 | 是 | 计收方式、分润、税务、收入确认 |
| 差错处理 | 视情况 | 视情况 | 视情况 | 状态多维组合、人工介入边界 |
| 投诉与争议处理 | 视情况 | 视情况 | 否 | 举证、时效、责任划分 |
| 监管/经营数据报送 | 无 | 否 | 否 | 报送时点、统计口径、失败重报 |

★ 第 0 轮明确"本期涉及哪些业务类型、不涉及哪些"，作为冻结基线传递。

## 3. 第三方接入全生命周期（第 1 轮）

第 1 轮需定位：本需求落在第三方合作生命周期的哪一段。不要默认是"日常交易"。

```
准入 → 协议签署 → 业务范围确认 → 清算账户确认 → 手续费规则确认
    → 接口联调 → 安全评估 → 参数配置 → 试点投产
    → 日常运营 → 对账清算 → 差错处理 → 服务监控
    → 合作方退出/暂停服务
```

第 1 轮输出模板（挂在 core-engine 第 1 轮"需求基本信息"之后）：

```

| 项 | 内容 | 原文证据 |

| 是否涉及合作机构准入审批 | 是/否/待确认 | |
| 是否涉及协议与清算账户确认 | 是/否/待确认 | |
| 是否涉及合作方退出或暂停 | 是/否/待确认 | |
| 不在本期的阶段 | | |
```

★ 准入、协议、清算账户、手续费规则若原文未明确，一律列待确认问题，确认部门标【待业务/财会/法务条线确认】。
